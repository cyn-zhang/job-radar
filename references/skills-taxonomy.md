## ATS Tips

1. **Mirror exact phrases** — if JD says "CI/CD pipelines", use that phrase not "deployment automation"
2. **Spell out acronyms once** — "Machine Learning (ML)" so both match
3. **Avoid tables/columns** for skills sections in DOCX — ATS parsers often miss text in table cells
4. **Use standard section headings** — "Experience", "Education", "Skills" (not "What I've Done")
5. **File format** — DOCX is safer than PDF for most Australian ATS systems (Seek, Workday, SmartRecruiters)
